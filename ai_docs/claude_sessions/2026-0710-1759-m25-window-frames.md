## Scope (Postgres-faithful)

- `{ROWS|RANGE|GROUPS} [BETWEEN <start> AND <end>]` with bounds
  UNBOUNDED PRECEDING/FOLLOWING, `<n>` PRECEDING/FOLLOWING, CURRENT
  ROW; single-bound shorthand normalizes to `BETWEEN ... AND CURRENT
  ROW` at parse.
- ROWS full; GROUPS full (offsets step peer groups; requires ORDER BY);
  RANGE only with UNBOUNDED/CURRENT ROW bounds — `RANGE <n> PRECEDING`
  rejected (PG 11's typed sort-key arithmetic; documented as Later).
- Frames honored by aggregate windows + FIRST/LAST/NTH_VALUE; ignored
  by ranking and LAG/LEAD (as PG). Empty frames → NULL, COUNT 0.
- `EXCLUDE NO OTHERS` accepted as no-op; other exclusions rejected.
- PG-worded validation: illegal bound pairs at parse ("frame starting
  from following row cannot reference current row", ...), GROUPS
  without ORDER BY, column refs in offsets ("argument of ROWS must not
  contain variables" — offsets must be row-independent), and runtime
  offset checks (non-null, integer, non-negative).

## Implementation

- `sql/ast.go`: FrameMode (Range/Rows/Groups), FrameBoundType (ordered
  so start-after-end checks read as comparisons), FrameBound,
  WindowFrame; `Frame *WindowFrame` on ExWindow; walkExpr walks frame
  offsets.
- `sql/parser.go`: `windowFrame`/`frameBound`/`checkFrameOffset`
  replace the old rejection in `windowOver`. Offset exprs parse via
  `expression()` — safe because bounds always end at the
  PRECEDING/FOLLOWING keyword, so BETWEEN's AND is unambiguous.
- `sql/window.go` (the core):
  - `resolveFrame` → `frameSpec` (bound types + offsets evaluated once
    per window in `computeWindow`, with a nil row — parse guarantees
    row-independence; bad offsets error even over zero rows).
  - `peerGroups` builds per-partition peer runs once (`groups`,
    `groupOf`); no ORDER BY → one group spanning the partition.
  - `frameBounds(fs, n, size, groups, groupOf)` → half-open `[lo, hi)`
    position range: ROWS by arithmetic, RANGE/GROUPS through groups
    (CURRENT ROW = current peer group; GROUPS offsets step groups).
    Offsets clamp to partition size first (int-overflow guard for
    absurd literals); `hi = max(lo, ...)` makes empty frames explicit.
  - The old FIRST_VALUE fast path, `assignFrameEnd`, and the aggregate
    section of `assignWindow` collapsed into two frame-driven
    assigners: `assignValueFrame` (first/last/nth over the frame,
    empty → NULL; nth's n still per-row, n<1 still errors) and
    `assignAggFrame` (UNBOUNDED PRECEDING start → one accumulator
    extends forward, hi is nondecreasing for every end-bound type, so
    the default frame stays O(rows); moving start → recompute per
    distinct frame with last-(lo,hi) memoization since peers share
    frames; sliding ROWS is O(rows × width) — min/max can't retract).
  - `windowText`/`boundText` render the canonical BETWEEN form for
    EXPLAIN.
- `sql/params.go`: bindExpr deep-copies ExWindow.Frame (the shallow
  `c := *n` shares the pointer — binding must not mutate the cached
  AST) and binds both offsets.
- `sql/describe.go`: frame-offset placeholders infer as TInt (same
  pattern as m24's lag offset inference).

## Gotchas / notes for next time

- Milestone numbering: 25 = window frames. Next arc starts at 26.
- Windows run over *post-filter* rows: `where id = 2` shrinks the
  partition to one row — bit me in a test expectation.

  `err.Error()`; the want/got detail rides in serr attributes. Test
  for "syntax error", not the wanted-token text.
